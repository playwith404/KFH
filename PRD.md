케이팝 피싱 헌터즈 (K-Phishing Hunterz)

**경찰청 사이버범죄 예방 공모전 제출용 기획서**

---

*피싱이 온다는 말은 너무 수동적이다. 내가 피싱에게로 간다.*

## **1. 프로젝트 개요**

### **1.1 핵심 컨셉**

> "피싱이 온다는 말은 너무 수동적이다. 내가 피싱에게로 간다."
> 

케이팝 피싱 헌터즈는 보이스피싱과 스캠을 **역으로 낚아** 사기꾼의 정보를 수집하고, 그들의 데이터베이스를 오염시켜 범죄 생태계의 경제성을 무너뜨리는 **시민 참여형 사이버 수사 협력 플랫폼**입니다.

### **1.2 문제 인식**

사기꾼들은 효율성을 극도로 따지는 집단입니다. 그러나 그들이 수집한 데이터베이스(DB)에 가짜 정보가 90% 이상 섞여버리면, 사기를 치기 위해 들이는 리소스가 결과(수익)보다 커지게 되어 결국 그 생태계가 무너집니다.

이를 '데이터 오염(Data Poisoning) 공격'이라고 합니다.

### **1.3 목표**

| **구분** | **1년차** | **3년차** |
| --- | --- | --- |
| 헌터 등록 | 10,000명 | 50,000명 |
| 미끼 배포 | 100,000건 | 500,000건 |
| 사기 계좌 신고 | 5,000건 | 25,000건 |
| 예상 피해 방지액 | 50억원 | 300억원 |
| DB 오염률 | 20% | 40% |

## **2. 법적 기반**

### **2.1 근거 법령**

| **법령** | **조항** | **적용 내용** |
| --- | --- | --- |
| 형사소송법 | 제199조 | 수사에 필요한 조사 활동 |
| 전기통신사업법 | 제83조 | 범죄 수사를 위한 통신자료 제공 |
| 개인정보보호법 | 제58조 | 범죄 수사 목적 시 적용 예외 |
| 전기통신금융사기 피해 방지 및 피해금 환급에 관한 특별법 | 전문 | 사기 계좌 지급정지 및 피해금 환급 |

### **2.2 헌터의 법적 지위**

**공식 명칭:** 사이버범죄 예방 시민협력단

**법적 보호:**

- 경찰청장 명의 위촉장 발급으로 공식 수사 협조자 지위 부여
- 활동 중 발생한 민사상 책임은 경찰청이 1차 부담
- 고의적 허위 신고 시에만 헌터 개인 책임
- 사기꾼 보복으로부터 신원 보호 (실명은 경찰 DB에만 보관)

**의무 사항:**

- 활동 전 온라인 교육 이수 (2시간)
- 서약서 작성 및 비밀유지 동의
- 분기별 보수교육 참여

### **2.3 합법적 활동 범위**

**허용되는 활동:**

- 가상 번호/ID를 통한 미끼 배포
- AI 페르소나를 통한 사기꾼과의 대화
- 사기 계좌/URL 정보 추출 및 신고
- 사기꾼 접속 환경(IP, OS) 추적

**제한되는 활동:**

- 사기꾼 시스템에 대한 직접적인 해킹이나 데이터 주입
- 헌터 개인 명의로 거짓 입금 확인 메시지 발송 (경찰 서버에서만 가능)
- 실명 또는 실제 개인정보를 이용한 활동

## **3. 서비스 스토리보드**

### **SCENE 1: 명예 케피헌 임명 (온보딩)**

**화면 구성:** 웅장한 디자인의 '명예 헌터 임명장' 연출

**사용자 동작:**

1. 실명 인증 (본인 확인)
2. 온라인 교육 이수 (2시간)
3. 피싱 대응 테스트 (10문항, 80점 이상 통과)
4. 서약서 전자 서명

**시스템 반응:**

- "OOO 님, 이제부터 대한민국 12,402번째 명예 케피헌입니다."
- 개인 고유 가상 번호 및 미끼 아이디 발급
- '헌터 자격증' 형태의 디지털 카드 발급

---

### **SCENE 2: 미끼 살포 (능동적 활동)**

**화면 구성:** 미끼 메뉴에서 배포 가능한 가짜 정보 리스트 확인

- 가상 전화번호
- 추적용 카카오톡 ID
- 위장 중고물품 정보
- 가짜 대출 희망 프로필

**사용자 동작:**

1. 미끼 유형 선택
2. [미끼 복사하기] 버튼 클릭
3. 중고 거래 커뮤니티, SNS, 커뮤니티 등에 미끼 정보 배포

**시스템 반응:**

- 미끼 배포 플랫폼 체크
- "낚시대를 던졌습니다. 사냥꾼의 연락을 기다리는 중..." 상태 표시
- 배포 포인트 +10P 적립

---

### **SCENE 3: 낚시 성공 (AI 실시간 개입)**

**화면 구성:** 스마트폰 푸시 알림 [헌터 모드 가동: 사기꾼 유입 감지! 🚨]

**사용자 동작:**

- 알림 클릭하여 '실시간 사냥 중계' 화면 진입
- 직접 통화하지 않고 관전 모드로 참여

**시스템 반응:**

- **좌측:** 스캐머의 채팅/음성 (STT로 텍스트화)
- **우측:** 케피헌 AI(페르소나: 어리숙한 노인)의 대응 메시지 실시간 노출
- **하단:** "사기 의심도 98%. 계좌번호 유도 중..." 상태바 진행
- 접촉 유도 성공 시 +50P 적립

---

### **SCENE 4: 정보 수집 및 신고 (Data Extraction)**

**화면 구성:** 사기꾼이 "신한은행 110-123..." 계좌를 보낸 순간

- 빨간색 강조 표시
- [정보 획득 성공] 이펙트 발생

**사용자 동작:**

1. 대화 전문 검토 (체크리스트 확인)
2. [신고 요청] 버튼 터치

**시스템 반응:**

- 정보 추출 성공 시 +100P 적립
- 3단계 검증 프로세스 진입 안내
- (경찰 서버에서) 사기꾼에게 시간 끌기용 메시지 자동 발송

---

### **SCENE 5: 3단계 검증 및 최종 신고**

**1차 검증 (AI 엔진):**

- 5개 사기 판정 지표 중 3개 이상 해당 여부 확인
- 자동 통과/보류 판정

**2차 검증 (헌터 검토):**

- 대화 전문 확인
- 8개 항목 체크리스트 작성
- 신고 근거 명시

**3차 검증 (경찰 승인):**

- 담당 수사관 최종 검토
- 금융결제원 및 경찰청 DB 전송 승인

**교차검증 요건:**

- 동일 계좌가 2명 이상의 독립된 헌터에게서 신고 시 우선 처리
- 단일 신고는 '모니터링 대상'으로 30일간 추가 신고 대기

---

### **SCENE 6: 전과 보고 및 보상 (Reward)**

**화면 구성:** 금일 사냥 리포트 발행

**내용:**

- "오늘 당신은 1건의 사기 계좌를 동결시켰고, 예상 피해액 500만 원을 방어했습니다."
- 획득 포인트 내역
- 헌터 등급 현황

**사용자 동작:**

- 포인트로 보상 교환
- 명예의 전당 랭킹 확인

**시스템 반응:**

- 최종 신고 승인 시 +300P 적립
- 실제 검거 기여 시 +1,000P 적립
- 등급 상승 조건 충족 시 승급 알림

---

### **SCENE 7: 데이터 오염 현황 (Global View)**

**화면 구성:** 전국 지도 화면

- 케피헌들이 뿌린 미끼에 낚인 사기꾼들의 위치(IP 기반) 실시간 도트 표시
- 지역별 활동 현황 히트맵

**표시 정보:**

- 전체 오염도: "현재 사기꾼들이 수집한 DB 중 42%가 케피헌의 가짜 정보로 오염되었습니다."
- 범죄 수익성 변화: "범죄 수익성이 전년 대비 15% 하락했습니다."
- 금일 전국 헌터 활동량

## **4. 오탐 방지 시스템**

### **4.1 3단계 검증 프로세스**

| **단계** | **주체** | **기준** | **통과 조건** |
| --- | --- | --- | --- |
| 1차 | AI 엔진 | 사기 패턴 매칭 | 5개 지표 중 3개 이상 해당 |
| 2차 | 헌터 검토 | 대화 전문 확인 | 체크리스트 8개 항목 확인 |
| 3차 | 경찰 승인 | 최종 신고 결정 | 담당 수사관 승인 |

### **4.2 사기 판정 5대 지표**

1. **선입금 요구:** 물품/서비스 제공 전 금전 요구
2. **안전결제 사칭:** 가짜 안전결제 링크 발송
3. **타 플랫폼 이동 유도:** 카카오톡, 텔레그램 등으로 대화 이동 요청
4. **급박한 시간 압박:** "오늘 안에", "지금 바로" 등 즉시 결정 유도
5. **신원 확인 회피:** 실명, 사업자등록번호 등 확인 거부

### **4.3 헌터 검토 체크리스트 (8개 항목)**

- [ ]  상대방이 먼저 연락해왔는가?
- [ ]  선입금을 요구했는가?
- [ ]  의심스러운 링크를 발송했는가?
- [ ]  다른 메신저로 이동을 유도했는가?
- [ ]  급하게 결정을 요구했는가?
- [ ]  신원 확인을 회피했는가?
- [ ]  계좌번호 또는 개인정보를 요구했는가?
- [ ]  대화 내용이 전형적인 사기 수법과 일치하는가?

### **4.4 교차검증 시스템**

- 동일 계좌/전화번호가 **2명 이상**의 독립된 헌터에게서 신고되어야 실제 수사 의뢰
- 단일 신고는 **'모니터링 대상'**으로 분류 후 30일간 추가 신고 대기
- 30일 내 추가 신고 없을 시 자동 종결 (허위 신고 방지)

### **4.5 오탐 발생 시 구제 절차**

1. **즉시 대응:** 오탐 확인 즉시 관련 기관에 동결 해제 요청
2. **피해 보상:** 오탐 피해자에게 경찰청 명의로 공식 사과 및 보상
3. **원인 분석:** 오탐 원인 분석 후 AI 엔진 및 체크리스트 개선
4. **헌터 조치:** 오탐 유발 헌터에 대한 경고/강등 조치

## **5. 미끼 유형 표준화**

### **5.1 미끼 설계 원칙**

**핵심:** 정상 거래자가 연락할 이유가 없는 구조로 설계

모든 미끼는 **'피해자 역할'**로 설계하여, 정상적인 판매자/구매자가 접근할 동기를 원천 차단합니다.

### **5.2 표준 미끼 유형**

**A유형: 급처 피해자형**

> "급전 필요해서 아이폰 15 Pro 50만원에 급처합니다. 직거래 어렵고 택배만 가능해요. 카톡 [미끼ID]"
> 
- **타겟:** 대리 판매 사기꾼
- **예상 접근:** "제가 대신 팔아드릴게요, 수수료 먼저 보내주세요"
- **정상 거래자 차단:** 판매자가 연락할 이유 없음

---

**B유형: 대출 희망자형**

> "신용등급 낮은데 500만원 정도 대출 가능할까요? 급해서 조건 안 따집니다. 연락처 [미끼번호]"
> 
- **타겟:** 불법 대출 브로커
- **예상 접근:** "수수료 먼저 입금하시면 당일 대출 가능합니다"
- **정상 거래자 차단:** 정상 금융기관은 이런 글에 연락하지 않음

---

**C유형: 투자 입문자형**

> "주식 처음인데 수익률 좋은 리딩방 있을까요? 월 100만원까지 투자 가능해요. 카톡 [미끼ID]"
> 
- **타겟:** 투자 사기 조직
- **예상 접근:** "VIP방 입장료 50만원입니다"
- **정상 거래자 차단:** 정상적인 투자 조언자는 이런 방식으로 접근하지 않음

---

**D유형: 로맨스 취약자형**

> "[소개팅 앱 프로필] 외로워요. 진지하게 만날 분 찾습니다."
> 
- **타겟:** 로맨스 스캠 조직
- **예상 접근:** "사업 자금이 필요해서..." "병원비가..."
- **정상 거래자 차단:** 페르소나를 통해 점진적으로 사기 유도 여부 확인

### **5.3 미끼 배포 가이드라인**

**권장 플랫폼:**

- 중고거래: 번개장터, 당근마켓 (사기 빈발 카테고리)
- 대출: 네이버 카페, 커뮤니티 대출 게시판
- 투자: 주식 커뮤니티, 텔레그램 오픈채팅
- 로맨스: 지정된 훈련용 플랫폼만 (민감도 높음)

**금지 사항:**

- 실제 판매자/구매자가 혼동할 수 있는 미끼 배포 금지
- 동일 플랫폼에 과도한 미끼 배포 금지 (24시간 내 3건 이하)
- 미끼에 실제 개인정보 포함 절대 금지

## **6. 다중 페르소나 AI 에이전트**

### **6.1 페르소나 유형**

| **페르소나** | **특성** | **적용 상황** |
| --- | --- | --- |
| 어리숙한 노인 | 귀가 어둡고 말이 느림, 같은 말 반복 | 보이스피싱 |
| IT 서툰 중장년 | 비밀번호 자주 틀림, 앱 설치 어려워함 | 메신저 피싱, 원격제어 사기 |
| 급한 직장인 | 빨리 처리하고 싶어함, 확인 절차 생략 요청 | 대출 사기 |
| 순진한 대학생 | 투자에 관심 많음, 의심 없이 질문 많이 함 | 투자 사기 |
| 외로운 중년 | 감정적 교류에 취약, 금전 요청에 쉽게 응함 | 로맨스 스캠 |

### **6.2 페르소나 자동 매칭**

사기 수법 감지 시 AI 엔진이 최적의 페르소나를 자동 선택:

1. **보이스피싱 감지** → '어리숙한 노인' 페르소나 활성화
    - 말을 길게 끌어 사기꾼의 통화 비용 증가 유도
    - "뭐라고요? 다시 말씀해주세요" 반복
2. **메신저 피싱 감지** → 'IT 서툰 중장년' 페르소나 활성화
    - 비밀번호를 계속 틀려 사기꾼 짜증 유발
    - "이 버튼이 어디 있어요?" 무한 반복
3. **투자 사기 감지** → '순진한 대학생' 페르소나 활성화
    - 수익률, 투자 방법 등 질문을 끊임없이 하여 시간 소모
    - 입금 직전에 "부모님께 여쭤볼게요" 지연

### **6.3 정보 추출 전략**

**목표 정보:**

- 입금 계좌번호
- 사기 조직 연락처 (전화번호, 메신저 ID)
- 피싱 URL
- 사기꾼 접속 환경 (IP, OS, 위치)

**추출 기법:**

- 자연스러운 대화 흐름 속에서 계좌번호 요청 유도
- "확인차 다시 한번 말씀해주세요" 기법으로 정보 재확인
- 사기꾼이 보낸 링크 클릭 시 Canary Token으로 역추적

## **7. 데이터 오염 전략**

### **7.1 합법적 오염 방식: 수동적 오염 전략**

**기존 문제점:** 사기꾼 DB에 직접 접근하여 데이터 주입 = 불법 해킹

**합법적 대안:** 사기꾼이 **스스로 수집해가도록 유도**

**프로세스:**

1. 헌터들이 대량의 가짜 프로필을 공개된 플랫폼에 게시
2. 사기꾼이 크롤링/수작업으로 해당 정보 수집
3. 수집된 DB에 가짜 정보가 자연스럽게 섞임
4. 사기꾼이 가짜 정보로 연락 시 AI가 응대하며 시간 낭비 유발

**법적 해석:** 정보를 공개된 곳에 게시한 것일 뿐, 적극적인 침입 행위 없음

### **7.2 오염 효과 측정**

**오염 성공 판정:**

- 헌터가 배포한 가짜 번호로 사기 시도가 들어오면 '오염 성공'으로 카운트
- 동일 가짜 프로필이 여러 사기 조직에서 사용될 경우 '광역 오염'으로 분류

**오염도 지표 산출:**

`오염도(%) = (가짜 정보로 인한 사기 시도 건수 / 전체 사기 시도 건수) × 100`

**목표 오염도:**

- 1년차: 20%
- 3년차: 40%
- 5년차: 60% (사기 비즈니스 손익분기점 붕괴 예상)

### **10.3 데이터 흐름**

`1. Baiting (미끼)
   헌터 → 플랫폼에서 가상 번호/ID 발급 → 사기꾼 소굴에 노출

2. Hooking (낚시)
   사기꾼 → 가상 번호로 연락 → AI 서버 0.1초 내 응답 시작

3. Engagement (교전)
   AI 페르소나 ↔ 사기꾼 대화 → 헌터 실시간 모니터링

4. Extraction (추출)
   사기꾼 "이 계좌로 입금" 발언 → 자동 추출 및 강조 표시

5. Verification (검증)
   추출 정보 → 1차 AI → 2차 헌터 → 3차 경찰 승인

6. Reporting (신고)
   승인된 정보 → 금융결제원 / 경찰청 DB 전송

7. Reward (보상)
   성공 건수에 따라 헌터 포인트 적립 및 등급 반영`

우리의 목적은 **방패를 만드는 것이 아니라, 사기꾼들의 화살촉을 뭉툭하게 만드는 가짜 과녁을 수천 개 만드는 것**입니다.
